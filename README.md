# AI Engineer Roadmap — Plan 1

## 🧠 Descripción

Esta organización reúne los proyectos del **Plan 1 — IA / Machine Learning Engineering**.

El objetivo de esta ruta es construir una base profesional como **AI Engineer / Machine Learning Engineer**, no solo entrenando modelos en notebooks, sino creando sistemas completos de IA:

```txt
datos → features → baseline → modelo → API → frontend → deploy → monitoreo → retraining → RAG/agentes → optimización
```

La idea central de este plan es aprender a construir IA real, documentarla, medirla, desplegarla y convertirla en evidencia visible de portafolio.

---

## 🎯 Objetivo general

Convertirme en un AI Engineer capaz de:

* Construir modelos de Machine Learning.
* Diseñar pipelines de datos.
* Crear APIs para servir modelos.
* Comparar modelos con métricas.
* Crear demos funcionales.
* Desplegar sistemas.
* Monitorear modelos.
* Entender MLOps.
* Construir RAG y agentes.
* Medir costos y latencia.
* Integrar todo en una plataforma final.

---

## 🧭 Filosofía de trabajo

Cada proyecto se trabaja con un ciclo de estudio de 4 días:

```txt
Día 1 → Exploración
Día 2 → Ejecución
Día 3 → Ejecución
Día 4 → Ejecución
```

### Día de exploración

Sirve para:

* Entender el tema.
* Investigar conceptos.
* Definir el mapa de los próximos 3 días.
* Identificar módulos y labs.
* Preparar el criterio técnico antes de construir.

### Días de ejecución

Sirven para:

* Implementar.
* Probar.
* Corregir errores.
* Documentar.
* Cerrar labs.
* Generar evidencia visible.

## 🧩 Conceptos base

### Proyecto principal

Un proyecto principal es un repositorio fuerte de GitHub.

Debe demostrar una habilidad grande y tener, según corresponda:

* Backend.
* Frontend.
* Modelo.
* API.
* Tests.
* README.
* Demo.
* Documentación.
* Labs.

### Módulo

Un módulo es una variación técnica dentro del proyecto.

Ejemplo:

```txt
Módulo: Model Comparison
```

Significa:

```txt
Comparar baseline vs Linear Regression vs Random Forest.
Medir.
Documentar.
Elegir con criterio.
```

### Lab

Un lab es un experimento pequeño y cerrado.

Regla personal:

```txt
Un lab debe durar 1 día de horario.
```

Un lab no debe quedar suelto. Debe conectarse al proyecto principal.

---

## 🧪 Tipos de labs

### tec-labs

Laboratorios técnicos.

Ejemplos:

* Feature Engineering.
* Model Comparison.
* Interpretability.
* Drift Detection.
* RAG Evaluation.
* Latency Benchmarking.

### cloud-labs

Laboratorios de traducción cloud.

Ejemplos:

* Render → Cloud Run.
* Local SQL → BigQuery.
* Model artifact → AWS S3.
* Model artifact → Azure Blob Storage.

### biz-labs

Laboratorios de negocio, usados más adelante en planes de empresa o producto.
